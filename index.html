⚠️ Key Risks & Observations

🔴 Performance: BFF 504 Gateway Timeouts Under Load High
Performance testing at 300 TPM surfaced HTTP 504 Gateway Timeout errors on ListPolicyDocuments, GetUserProfile, and GetUserPolicies. GetPolicyDetail shows worst-case response times of ~31 s. Root cause traced to backend integration services hanging at the CIL layer before responding. Sequential upstream calls in GetPolicy (5 provider round-trips) compound latency.
CIL-5085 In Code Review CIL-5394 Parallel calls In Progress
🔴 Blocker: Lookup Value Normalisation (CHOE) Blocker
OE-formatted lookup values (e.g. BUSINESS) cannot be directly sent back via Update Insured Person — CIL expects its own ENUM format (e.g. WORK). Affects contact type, payment frequency, marital status lookups. Without normalisation at response time the frontend has no deterministic way to map values, breaking Update Insured Person flows.
